General System Disclosure

LOAN XAI SYSTEM is built on the philosophy of complete algorithmic transparency. In traditional financial credit underwriting, credit scores are produced by closed proprietary formulas, often leaving applicants and bank operators in the dark about why a loan was rejected.

This platform exposes all levels of risk scoring:

  • Decoupled Architecture: A fast Python client layer (Flask) provides visual representations, while a high-performance Python FastAPI engine handles the math, data scaling, and model prediction in-memory.
  • Explainable Decisions: Instead of simple yes/no outputs, every prediction is accompanied by a game-theoretic decomposition (SHAP values) that shows which personal financial traits helped or hurt the applicant's credit score.
  • Zero-Database Inference: The core pipeline is stateless. Data is processed in-memory for security and speed, and logged locally for compliance auditing.

Explainability Math & Surrogates

Our system utilizes two primary algorithms to verify prediction outcomes. The primary tree-based model (LightGBM) is paired with SHAP, while the academic notebooks cross-validate using LIME.

1. SHAP (SHapley Additive exPlanations)

SHAP is grounded in cooperative game theory. In this context, features of a loan applicant are treated as "players" collaborating to obtain the prediction value \(f(x)\). The Shapley value \(\phi_i\) represents the fair payout (contribution) attributed to feature \(i\).

It models prediction output as a linear addition of feature effects:

\[g(z') = \phi_0 + \sum_{i=1}^{M} \phi_i z'_i\]

Where \(z' \in \{0, 1\}^M\) indicates the presence (1) or absence (0) of a feature subset, and \(\phi_0\) is the average model output across all training profiles. The exact marginal contribution is formulated as:

\[\phi_i = \sum_{S \subseteq F \setminus \{i\}} \frac{|S|!(|F| - |S| - 1)!}{|F|!} \left[ f_x(S \cup \{i\}) - f_x(S) \right]\]

For our LightGBM model, computing this exactly would be computationally prohibitive. Therefore, we utilize **TreeExplainer**, which traverses the paths of the tree structures to compute exact Shapley values in polynomial time:

\[O(T L D^2)\]

Where \(T\) is the number of trees, \(L\) is the maximum leaves, and \(D\) is the maximum depth.

2. LIME (Local Interpretable Model-agnostic Explanations)

Unlike SHAP's global game-theoretic foundation, LIME is model-agnostic and relies on local linear surrogates. To explain a complex model's prediction at \(x\), LIME:

  1. Perturbs the input data \(x\) slightly to create a synthetic neighborhood dataset.
  2. Computes prediction outputs from the black-box model \(f\) for all perturbed data points.
  3. Weights the perturbed data points using a similarity/proximity kernel \(\pi_x(z)\) (typically a exponential Gaussian kernel).
  4. Fits an interpretable linear surrogate model \(g\) by minimizing the weighted squared loss:
\[\text{Explanation}(x) = \arg\min_{g \in G} L(f, g, \pi_x) + \Omega(g)\]

Where \(L(f, g, \pi_x) = \sum_{z, z'} \pi_x(z) (f(z) - g(z'))^2\) and \(\Omega(g)\) is the penalty on model complexity (e.g. LASSO regularization).

Model Architecture Comparison

The automated training pipeline evaluates several different machine learning paradigms to establish the optimal underwriting decision boundary:

Model Class Algorithm Details Why We Used It
LightGBM Leaf-wise tree growth gradient boosting. Hyperparameters optimized using Bayesian tuning: learning rate 0.03, num_leaves 31. Highly efficient, handles mixed numerical/categorical scales natively, and supports TreeExplainer for fast SHAP outputs.
XGBoost Level-wise tree growth with depth limits. Applies regularization (L1/L2) inside tree leaf weighting. Provides robust regularization against outliers and serves as a cross-verification boosting baseline.
Tabular MLP Multi-Layer Perceptron (Keras). Dense layer configurations with Dropout layers (rate: 0.3) and learning rate scheduling. Explores deep non-linear interaction spaces that standard trees might miss. Requires standard scaler.
Soft Voting Ensemble Probability-weighted combination of LightGBM, XGBoost, and MLP. Combines predictions by averaging probability scores to reduce variance and improve model generalization.
Logistic Regression Linear classifier with L2 weight regularization (Ridge). Simplest linear model baseline. Standard feature scaling is integrated directly into the pipeline.

In-Memory Feature Transformations & Scaling

The machine learning models expect a 36-dimensional matrix. Raw form inputs consist of only 16 parameters. The [XAIEngine](file:///home/sk/Videos/Karma/depi_loan_default_xai/backend_api/app/xai_engine.py#L19) constructs 27 derived interaction features in-memory before running the scaler:

1. Interaction Equations:

  • Estimated Monthly Payment: \[\text{monthly\_payment\_est} = \frac{\text{loanamount} \times \frac{\text{interestrate}}{100}}{\text{loanterm} + 1}\]
  • Affordability Score: \[\text{payment\_to\_income} = \frac{\text{monthly\_payment\_est}}{\frac{\text{income}}{12} + 1}\]
  • Stress Index: \[\text{debt\_burden\_score} = \text{dtiratio} \times \text{interestrate}\]
  • Composite Risk Metric: \[\text{risk\_score} = 0.40 \times \left(1 - \frac{\text{creditscore} - 300}{550}\right) + 0.35 \times \text{dtiratio} + 0.25 \times \frac{\text{interestrate}}{30}\]

2. Feature Scaling Mechanics

For linear baseline models (Logistic Regression) and neural networks (MLPs), variables with large scales (such as Income or Loan Amount) would dominate gradient weights. Therefore, we fit a StandardScaler on the training features.

Each feature value \(x\) is transformed by subtracting its training mean \(\mu\) and dividing by its standard deviation \(\sigma\):

\[z = \frac{x - \mu}{\sigma}\]

Note: Tree-based models like LightGBM and XGBoost are invariant to monotonic transformations and do not require scaling to calculate splits, but the standardized scaling is maintained globally for consistency in the pipeline.
