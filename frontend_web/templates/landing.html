LOAN XAI SYSTEM

Automated Explainable AI for Loan Default Prediction

Project Summary

This project implements an end-to-end automated machine learning pipeline for predicting loan defaults, with a strong emphasis on model explainability and interpretability.

  • ⚡
    Multi-Model Training Pipeline: LightGBM, XGBoost, Random Forest, Logistic Regression, Ensemble Soft Voting, and deep learning Tabular MLPs (Keras).
  • 📊
    SHAP & LIME Explainability: Generates SHAP waterfall charts, beeswarm plots, alongside LIME local surrogate models for prediction transparency.
  • 🎯
    Automated Feature Engineering: Computes 27+ engineered interaction features dynamically for the 36-dimensional feature space.

System Architecture & Project Documentation

01

ML Pipeline Stages

Stage 1

Exploratory Data Analysis (EDA)

notebook/EDA.ipynb

Loading the raw Kaggle Loan Default dataset and performing a thorough statistical exploration. Includes computing summary statistics, plotting distribution histograms to identify skewness, and generating a correlation heatmap to detect multicollinearity.

Stage 2

Data Preprocessing

notebook/Preprocessing.ipynb

Handles data cleaning and transformation. Operations include handling missing values, encoding categorical variables, applying log transformations, fitting a StandardScaler, and splitting the dataset into training, validation, and test partitions.

Stage 3

Model Building & Evaluation

notebook/building_models.ipynb

Training multiple models including Logistic Regression, Random Forest, LightGBM (Base, SMOTE, Tuned, Early Stopping), XGBoost, Ensemble Soft Voting, and Tabular Deep Learning MLP. Each model is evaluated using ROC-AUC curves and precision-recall analysis.

Stage 4

SHAP & LIME Explainability

notebook/shap_lime_explainer.ipynb

Applies post-hoc explainability techniques. SHAP TreeExplainer computes Shapley values for generating waterfall and beeswarm plots. LIME fits local linear surrogate models around individual predictions to cross-validate feature importance.
